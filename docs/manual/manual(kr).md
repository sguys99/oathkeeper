## 1. 개요

OathKeeper 웹 인터페이스는 5개의 주요 네비게이션 메뉴로 구성됩니다:

| 메뉴 | 경로 | 설명 |
|------|------|------|
| **분석 요청** | `/` | 새로운 Deal의 AI 분석 요청 |
| **Deal 현황** | `/deals` | 전체 Deal 조회 및 관리 |
| **에이전트 로그** | `/agent-logs` | 에이전트 실행 이력 확인 |
| **에이전트 설정** | `/agent-settings` | 에이전트 프롬프트 및 플로우 설정 |
| **관리자** | `/admin` | 회사 정보, 평가 기준, 인력, 비용, 프로젝트 이력 관리 |

## 2. 분석 요청

**경로:** `/`

Deal의 AI 기반 Go/No-Go 분석을 요청하는 메인 페이지입니다.

![분석 요청 페이지](imgs/1.request-analysis.png)

### UI 요소

| 요소 | 설명 |
|------|------|
| **Notion Deal 선택** | 드롭다운에서 연동된 Notion 데이터베이스의 Deal을 선택합니다. |
| **추가 정보** *(선택)* | Deal에 대한 보충 설명, 요구사항, 참고 사항을 자유롭게 입력할 수 있습니다. |
| **문서 첨부** *(선택)* | `.docx` 또는 `.pdf` 파일을 드래그 앤 드롭하거나 클릭하여 업로드합니다 (최대 20MB). 첨부된 문서는 파싱되어 분석에 포함됩니다. |
| **분석 시작** | 분석 파이프라인을 시작합니다. SSE를 통해 각 단계의 진행 상황이 실시간으로 표시됩니다. |

> **참고:** 이미 분석된 Deal을 선택하면 중복 경고와 함께 기존 분석 결과 링크가 표시됩니다.

## 3. Deal 현황

**경로:** `/deals`

전체 Deal을 검색, 필터링하고 관리할 수 있는 대시보드입니다.

![Deal 현황](imgs/2-1.deal-info-overview.png)

### UI 요소

| 요소 | 설명 |
|------|------|
| **검색바** | Deal 제목으로 실시간 검색합니다. |
| **상태 필터** | 드롭다운으로 상태별 필터링: `all`, `pending`, `analyzing`, `completed`, `failed`. |
| **Deal 테이블** | **고객사/Deal**, **판단**(배지), **점수**, **생성일**, **담당자** 컬럼을 표시합니다. |
| **+ 새 분석** | 우측 상단 버튼 — 분석 요청 페이지로 이동합니다. |
| **삭제** | 각 행의 휴지통 아이콘 — 확인 다이얼로그 후 Deal을 삭제합니다. |
| **행 클릭** | 해당 Deal의 상세 분석 결과 페이지로 이동합니다. |
| **페이지네이션** | 20건 단위로 페이지가 나뉩니다. |

## 4. Deal 상세

**경로:** `/deals/:id`

개별 Deal의 분석 결과를 종합적으로 확인하는 페이지입니다.

![Deal 상세](imgs/2-2.deal-info-detail.png)

### 종합 평가

| 요소 | 설명 |
|------|------|
| **종합 점수** | 0~100점의 전체 점수와 컬러 코딩 표시. |
| **판단 배지** | `Go` (≥ 70), `조건부 Go` (40–69), `No-Go` (< 40), 또는 `Hold`. |
| **레이더 차트** | 7개 평가 기준에 대한 시각적 비교 차트. |

### Deal 개요 (사이드바)

구조화된 Deal 메타데이터를 표시합니다: 고객사, 산업, 계약 금액, 수행 기간, 기술 스택.

### 평가 기준별 점수

7개 기준 각각에 대해 다음을 표시합니다:
- 수평 점수 바 (성과에 따른 컬러 코딩)
- 가중치 비율
- 한줄 평가 요약

### 추가 섹션 *(스크롤 아래)*

| 섹션 | 설명 |
|------|------|
| **리소스 산출** | 예상 팀 배치, 비용, 일정. |
| **리스크 분석** | 식별된 리스크와 심각도, 상호 의존성. |
| **유사 프로젝트** | 벡터 검색으로 찾은 상위 3개 유사 과거 프로젝트. |
| **추천사항** | 마크다운으로 렌더링된 전략적 권고 사항. |

### 액션 버튼

| 버튼 | 설명 |
|------|------|
| **Notion에 저장** | 분석 결과를 Notion 데이터베이스에 내보냅니다. |
| **로그 보기** | 해당 Deal의 에이전트 실행 로그 페이지로 이동합니다. |
| **목록으로** | Deal 현황 페이지로 돌아갑니다. |

## 5. 에이전트 로그

### 5.1 로그 목록

**경로:** `/agent-logs`

분석이 완료되었거나 실패한 모든 Deal의 목록을 표시합니다.

![에이전트 로그 목록](imgs/3-1.agent-log-overview.png)

각 카드에 표시되는 정보:

| 요소 | 설명 |
|------|------|
| **Deal 제목** | 분석된 Deal의 이름. |
| **타임스탬프** | 분석 일시. |
| **상태 배지** | `완료` 또는 `실패`. |
| **로그 보기** | 상세 실행 로그를 확인하는 버튼. |

---

### 5.2 로그 상세

**경로:** `/deals/:id/logs`

특정 Deal에 대한 에이전트 실행 파이프라인의 상세 뷰입니다.

![에이전트 로그 상세](imgs/3-2.agent-log-logs.png)

실행 과정은 **3단계 타임라인**으로 표시됩니다:

| 단계 | 노드 | 설명 |
|------|------|------|
| **1. 딜 구조화** | `deal_structuring` | 원본 Deal 입력을 구조화된 데이터로 파싱합니다. |
| **2. 병렬 분석** | `scoring`, `resource_estimation`, `risk_analysis`, `similar_project` | 4개 노드가 동시에 실행되어 Deal을 평가합니다. |
| **3. 최종 판단** | `final_verdict` | 모든 분석을 종합하여 Go/No-Go 결정을 내립니다. |

각 노드 카드에 표시되는 정보:
- **실행 시간**과 **상태** (성공 / 실패)
- **시스템 프롬프트** — LLM에 전달된 지시사항
- **유저 프롬프트** — 태스크별 입력 데이터
- **LLM 원본 출력** — 모델의 전체 응답
- **파싱된 결과 (JSON)** — 응답에서 추출한 구조화된 출력

모든 섹션은 접기/펼치기가 가능합니다.

## 6. 에이전트 설정

**경로:** `/agent-settings`

각 에이전트 노드를 구동하는 프롬프트 템플릿을 설정합니다.

![에이전트 설정](imgs/4.agent-settings.png)

### UI 요소

| 요소 | 설명 |
|------|------|
| **프롬프트 선택** | 드롭다운에서 프롬프트 템플릿 선택: `system`, `deal_structuring`, `scoring`, `resource_estimation`, `risk_analysis`, `similar_project`, `final_verdict`. |
| **버전 / 이름** | 프롬프트의 버전과 내부 이름을 표시하는 메타데이터 필드. |
| **설명** | 선택된 프롬프트의 역할과 목적을 설명합니다. |
| **시스템 프롬프트** | LLM 지시사항 편집 영역. Jinja2 템플릿 변수를 지원합니다 (예: `{{ company_context }}`, `{{ deal_criteria }}`). |
| **유저 프롬프트** | 태스크별 템플릿 편집 영역. |
| **에이전트 플로우 다이어그램** | 우측에 3단계 실행 파이프라인을 시각화한 플로우차트 (딜 구조화 → 보류 판정 → 병렬 분석 → 최종 판단). |
| **저장** | 프롬프트 변경 사항을 데이터베이스에 저장합니다. |

## 7. 관리자 설정

**경로:** `/admin`

5개 탭으로 구성된 중앙 설정 대시보드입니다.

![관리자 설정 — 평가 기준](imgs/5.admin.png)

### 탭 구성

| 탭 | 설명 |
|----|------|
| **회사 정보** | 사업 방향, 단/중/장기 전략, Deal 선정 기준 편집. |
| **평가 기준** | 7개 평가 기준의 가중치 조정 (합계 100%). |
| **인력 관리** | 팀원 추가, 수정, 삭제 및 역할/시급 설정. |
| **비용 설정** | 비용 항목(인프라, 도구, 서비스) 관리 및 단가/수량 설정. |
| **프로젝트 이력** | 과거 프로젝트 임베딩 관리 및 Pinecone 벡터 스토어 동기화. |

### 7.1 회사 정보

AI 에이전트가 분석 시 참고하는 회사 비즈니스 컨텍스트를 편집합니다:
- **사업 방향** — 회사의 전반적인 방향성과 중점 분야
- **단/중/장기 전략** — 시간대별 전략 계획
- **Deal 선정 기준** — 수주 Deal 평가 가이드라인

동작: **저장**으로 변경 사항을 반영하거나, **기본값으로 저장**으로 YAML 기본값을 복원합니다.

### 7.2 평가 기준

각 평가 기준의 가중치를 설정합니다. 위 스크린샷과 같이 슬라이더 인터페이스를 제공합니다:

| 기준 | 기본 가중치 |
|------|-----------|
| 기술 적합성 | 20% |
| 수익성 | 20% |
| 리소스 가용성 | 15% |
| 납기 리스크 | 15% |
| 요구사항 명확성 | 10% |
| 전략적 가치 | 10% |
| 고객 리스크 | 10% |

각 기준은 **이름**, **설명**, **가중치 슬라이더**를 표시합니다. 전체 합계는 반드시 **100%**여야 합니다.

### 7.3 인력 관리

리소스 산출에 사용되는 팀원 명단을 관리합니다:
- 이름, 역할, 시급을 입력하여 새 팀원 **추가**
- 기존 팀원 정보 **수정**
- 더 이상 소속되지 않은 팀원 **삭제**

### 7.4 비용 설정

리소스 산출에 반영되는 비용 항목을 설정합니다:
- 인프라, 도구, 라이선스, 서비스 비용 항목 관리
- 각 항목의 **단가**와 **수량** 설정

### 7.5 프로젝트 이력

유사 프로젝트 검색에 사용되는 과거 프로젝트 데이터를 관리합니다:
- 업로드된 프로젝트 기록 조회 및 관리
- **임베딩 새로고침** — 벡터 임베딩을 재생성하고 Pinecone과 동기화
